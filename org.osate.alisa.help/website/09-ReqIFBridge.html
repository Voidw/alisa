ReqIF bridge

  • Implementation Considerations and Requirements
  • Mapping Rules
  • Importing ReqIF requirements to ReqSpec
  • Exporting ReqSpec requirements to ReqIF

Alisa tools provide a bridge to import/export requirements designed with the ReqIF format. ReqIF is a standardized notation to capture requirements. It is supported by many tools, either commercial or open-source. A ReqIF bridge provides the ability to connect ALISA requirements with other tools.

This document details the mapping rules and how to import or export requirements from/to ALISA/ReqSpec to/from ReqIF.

Top

Implementation Considerations and Requirements

The bridge is built on top of ReqSpec and the Eclipse Requirements Modeling Framework (RMF). RMF is an Eclipse-based implementation of ReqIF and specifies an EMF meta-model for the ReqIF interchange format.

As both have their meta-models define with EMF, it facilitates to translation from one notation to the other. To be able to use the ALISA/ReqIF bridge, you need to have the RMF plugins installed in your Eclipse environment.

Importing ReqIF requirements to ReqSpec

  • Tailoring the Import Process

To import a ReqIF document, you have to open it using within Eclipse using the RMF tool.

The first step is to select the ReqIF specification hierarchy to export in the outline view, as shown below. The bridge can import only one specification at a time.

Selecting the ReqIF Specification to import in the Outline view

Once you selected the specification to import, select the ReqIF import menu in the ALISA menu as shown below. Note that two import options are available:

  1. System Requirements: imports ReqIF requirements as system requirements. This option will then map the reqif requirements into requirement object
  2. System Requirements: imports ReqIF requirements as stakeholders requirements. This option will map the reqif requiremnts into stakeholders requirements - goals

Invoking the tool

Once the import is completed, the tool created a new ReqSpec file, called either doors-import-systems.reqspec or doors-import-stakeholders.reqspec, depending on the type of import you are using.

Tailoring the Import Process

When importing a ReqIF document, the bridge will then use the attributes values from the SpecObjects from ReqIF to fill the ReqSpec model. ReqIF lets you defines your own specification model so that each ReqIF model can have a different way to capture ReqSpec model. For each requirement in ReqIF, there is a set of attributes identified by a long name. This long name is defined in the data types of the ReqIF document. To see the data types of your ReqIF document, you can use RMF and open the data type definition. To do so, open the requirements document with RMF and click on the following icons.

Icon to open the data types definition of the ReqIF document

Once you open the data types definition, you can see the long name by selecting the type. See below.

Preferences showing the long name of a requirement attribute

In order to adapt the bridge to each ReqIF specification model, the bridge has a preference mechanism for the user to specify how to associate an an ReqIF attribute with a ReqSpec attribute.

This preference mechanism is available within the Eclipse preferences, as shown below. For each property on ReqSpec object, the user can specify the corresponding ReqIF attribute. The attribute is designated by its long name. If you do not specify an identifier attribute, the bridge will then try to build identifiers automatically. If you specify a mapping between ReqIF and ReqSpec, you have to make sure that identifiers are unique in the document.

Finally, you add a prefix to the identifier in the preferences. It can help to distinguish stakeholders and system requirements.

To export the ReqSpec requirement into ReqIF, you first need to select the ReqSpec document in the outline view as shown below.

Selecting the ReqSpec document to export in the Outline view

Once the main ReqSpec object has been selected, you invoke the tool by selecting ReqIF Export in the ALISA tool menu as shown below.

Invoking the Export tool

Once the export is completed, a new file called reqspec-export.reqif appears in the project that contains the imported ReqSpec file.
